## Parsed requirements

### FB-1: Import set & cards
- User can import a flashcard set (and its cards) from file.
- Scope TBD: CSV/JSON minimum; validate rows; map to set + cards.

### FB-2: Study session settings & enhanced flow
- Pre-study settings screen before starting:
  - Choose study mode (including multiple choice).
  - Toggle randomize card order.
  - Session size: user-configurable batch (e.g. 7–10 cards per round).
- Spaced retry within study flow: cards answered wrong in an earlier batch are re-queued in later batches of the same study run.

### FB-3: Auth — forgot password email & session persistence
- Forgot password should send email (user reports no email received).
- Closing browser and returning should not force re-login (session should persist).

## Implementation notes (current codebase)

- **Forgot email**: `sendPasswordResetEmail` logs link to console when `RESEND_API_KEY` unset (`src/server/email.ts:16-18`). Production requires Resend key + verified sender domain.
- **Session**: JWT `maxAge` 30 days in `auth.config.ts`. Re-login after reopen may be caused by dev port switching (3000↔3002), missing `RESEND`/env, or cookie `maxAge` not propagated — needs verification.
- **Multiple choice**: exists in Learn/Test modes; user may want unified pre-study picker + MC as primary mode on `/study` or set detail.
- **Import**: not implemented in v1 tasks.
